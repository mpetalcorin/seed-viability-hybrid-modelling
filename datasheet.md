# Datasheet for the Simulated Seed Viability Dataset## MotivationThis dataset was created to support a computational proof-of-concept for modelling seed viability decline under storage. The aim was to provide a transparent, literature-benchmarked synthetic dataset that could be used to develop and test analytical workflows before application to real seed-lot measurements.## CompositionThe dataset is longitudinal. Each record represents a seed lot observed at a particular storage time point. Multiple repeated observations are therefore available for each lot.The dataset includes simulated seed lots spanning three storage behaviour categories:- orthodox- intermediate- recalcitrantFor each lot-time combination, the dataset includes environmental, biological, and derived outcome variables.## Data fieldsTypical variables in the dataset include:- `lot_id`, unique seed-lot identifier- `seed_class`, storage behaviour class- `month`, storage duration- `temperature_c`, storage temperature in degrees Celsius- `moisture_content_pct`, seed moisture content percentage- `water_activity`, proxy for storage humidity or moisture activity- `oxygen_level`, storage oxygen condition- `packaging`, storage or packaging condition- `initial_viability`, starting viability level- `initial_dormancy`, starting dormancy tendency- `viability`, simulated probability or proportion of viable seeds- `germination`, simulated probability or proportion of germinated seeds- `dormancy`, simulated probability or proportion of dormant seeds- `event_viability_below_50`, binary indicator for threshold failure- `mechanistic_prediction`, baseline viability prediction- `hybrid_prediction`, residual-corrected viability predictionSome variable names may differ slightly depending on the exact notebook version, but the structure follows this logic.## Data generation processThe dataset was generated by simulation rather than direct measurement. The simulation was constructed to produce biologically plausible behaviour consistent with broad findings from the seed science literature.The generation process included these steps:1. Assign each lot to a seed storage behaviour class.2. Sample initial lot properties such as viability and dormancy tendency.3. Sample storage conditions such as temperature, moisture, oxygen environment, and packaging condition.4. Simulate deterioration over time with faster decline under more stressful environments.5. Apply class-specific penalties so that recalcitrant seeds deteriorate more rapidly than intermediate or orthodox seeds.6. Derive germination and dormancy trajectories separately from viability.7. Define an event when viability drops below 50 percent.8. Create model-ready tables for regression and survival-style analysis.## Source dataNo real seed-lot observations are included in the dataset. The data are fully synthetic. Parameter ranges and qualitative relationships were inspired by peer-reviewed literature on seed longevity, seed storage behaviour, and oxygen-dependent ageing.## Collection processBecause the dataset is simulated, there was no physical collection process, no instrumentation, and no sampling from living organisms or field sites. The dataset exists solely as a computational construct for proof-of-concept analysis.## PreprocessingThe notebook performs feature engineering and preprocessing for modelling. This may include encoding categorical variables, scaling or transforming numeric variables where appropriate, creating event labels, and splitting the data into training and test partitions.## Recommended usesThis dataset is appropriate for:- testing notebook workflows- demonstrating survival-style seed modelling- teaching interpretable machine learning in seed science- prototyping visualization pipelines- building and comparing proof-of-concept models## Non-recommended usesThis dataset should not be used for:- species-specific biological inference- operational seed-bank decision-making- regulatory or commercial claims- benchmarking empirical storage performance- replacing experimental validation## DistributionThe dataset is distributed within the notebook as generated code rather than as a fixed external raw file. Users reproduce it by running the notebook cells.## Uncertainty and noiseThe simulation includes variability and noise to avoid a perfectly deterministic dataset, but the uncertainty structure is simplified relative to real seed biology. Real-world datasets may show stronger batch effects, measurement error, censoring, nonlinear threshold effects, or unrecorded confounders.## Biases and limitationsThe dataset encodes the assumptions of its designers. It reflects simplified literature-benchmarked relationships and may overrepresent smooth monotonic deterioration relative to real biological systems. It also may not capture abrupt threshold phenomena, species-specific dormancy physiology, maternal environment effects, pathogen burden, or post-harvest handling artifacts.## Relationship to the modelThe dataset was built specifically for the seed viability proof-of-concept notebook. It supports mechanistic baseline modelling, hazard-style event prediction, and hybrid residual learning. Because the dataset and model were co-designed, performance results should be interpreted as internal proof-of-concept rather than out-of-sample validation.## Maintenance and versioningIf the notebook changes, the generated dataset may also change. Users should therefore report the notebook filename, date, and repository commit hash when publishing results based on a specific version.## AcknowledgmentWhen reusing this synthetic dataset or adapting its structure, acknowledge that it was created as a literature-benchmarked computational proof-of-concept for seed viability modelling.
